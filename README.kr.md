# 🌌 MONO (Monad Object Notation) 공식 스펙 명세서

> 
> **"타입(Type)을 파괴하고, 오직 경로(Path)와 맥락(Context)으로 세상을 일원화한다."** > MONO는 기존 JSON/YAML의 타입 검사 피로감과 복잡성을 극복하고, 객체 접근 표현 방식(`object.property`)을 정의 시점부터 직관적으로 계승한 **초경량 무타입 데이터/문서 표기 표준**입니다.
> 
> 

---

## 🏛️ 1. MONO의 3대 철학 (Philosophy)

1. 
**무타입 일원화 (Pure Typelessness)** 


* 세상에 `String`, `Int`, `Boolean` 등의 고정 타입은 존재하지 않습니다.


* 모든 데이터는 오직 '연결(Link)'과 '맥락(Context)'으로만 존재합니다 (`25`는 숫자가 아닌, `age` 뒤에 연결된 데이터 조각일 뿐입니다).




2. 
**구조적 제약 방어 (Structural Defense)** 


* 내용물(Type)을 검사하는 컴파일 타임의 통제를 거부합니다.


* 트리의 '외형 골격(Shape)'과 '진입 경로(Constraint)'만으로 시스템의 카오스(Chaos)를 완벽히 방어합니다.




3. 
**코드와 데이터의 일원화 (Homoiconicity)** 


* 데이터 구조와 실행 명령어의 구조가 100% 동일하여, 데이터 자체가 곧 실행 가능한 프로그램이 됩니다 (`( run.print, print.message.Hello )`).

## ✍️ 2. 표기법 스펙 (Syntax & Notation)

상황과 용도에 맞춰 상호 호환되는 **하이브리드 포맷**을 지원합니다.

### ① 블록 표기법 (Block Notation - `.mn`)

* 
**용도:** 개발용 파일 작성, 인간 중심 가독성.


* 
**규칙:** 괄호 `()`와 쉼표 `,`를 생략하고, 줄바꿈(Newline)과 **탭(Tab) 들여쓰기**로 계층(Depth)을 표현합니다.


* 
**모호성 진압:** 공백이나 마침표(`.`)가 포함된 값은 따옴표 `""`로 감싸 파서가 원시 문자열로 흡수하게 만듭니다.



```mono
user
	#id	alice
	name	"Hong Gil Dong"
	skills
		Python
		JS

```

### ② 인라인 표기법 (Inline Notation)

* 
**용도:** 초경량 네트워크 패킷 전송, 한 줄 설정.


* 
**규칙:** 괄호 `()`와 쉼표 `,`를 활용한 튜플 노드로 표현합니다 (`( user.#id.alice, user.name."Hong Gil Dong" )`).

## 🛡️ 3. 데이터 적재 & 쿼리 헌법 (Data & Query Specs)

① 모든 데이터의 맵(Map)화 및 오토 인덱싱 (Implicit Index) 

* 
**경로 고유성 헌법:** 모든 데이터 조각은 메모리 상에서 단 하나의 고유 경로(Deterministic Path)를 가집니다.


* 
**중복 처리:** 동일 경로에 중복 입력된 리스트 형태는 파서 내부에서 `user.skills[0]`, `user.skills[1]`과 같이 순서 인덱스가 강제 결합된 맵(Map)으로 변환 저장됩니다.



② 명시적 키 vs 순서 인덱스 쿼리 구별법 (`a.1` vs `a[1]`) 

단순 숫자가 키값인지 인덱스 위치인지를 명확히 구분하여 파싱 모호성을 $100\%$ 진압합니다.

| 쿼리 표현식 | 동작 방식 | 예시 (`a` 하위에 명시적 키 `1`과 단말 노드 `d`가 있는 경우) |
| --- | --- | --- |
| **`a.1`** | 명시적 키/경로 이름이 **`"1"`인 자식 노드** 직접 조회 

 | <br>`1` 반환 

 |
| **`a[1]`** | 이름과 무관하게 **순서상 1번 인덱스(두 번째 자식)** 위치 조회 

 | <br>`d` 반환 

 |

```javascript
// JavaScript Mono 클래스 활용
const data = new Mono("a((b,c), d)");
data.get("[0].[0]").value; // "b"
data.get("[1]").value;    // "d"

```

③ 서브 트리(Sub-tree) 반환 법칙 

* 중간 경로(예: `get("user.skills")`)를 쿼리할 경우 에러를 발생시키지 않고, 해당 지점을 루트로 하는 하위 인덱스 맵 전체(Sub-tree)를 반환합니다.


* 
**오류 발생 조건:** 오직 존재하지 않는 경로를 요청했거나 필수 제약(`!`)을 위반했을 때만 에러를 유발합니다.

🏷️ 4. 방어막 토큰 (Guard Tokens) 

데이터의 성격과 불변성을 제어하기 위해 특수 기호를 접두어로 부여합니다.

* 
`!` **(필수 제약):** 해당 노드 하위에 반드시 값이 존재해야 함을 강제.


* 
`#` **(유니크/상수 제약):** 단일 Key 기반의 고유 공간(Map)임을 나타내며, 중복 및 불변성을 지킵니다.


* 
`@` **(태그/스타일 속성):** 노드에 독립된 스타일 차원이나 메타 속성을 부여합니다.



---

🎨 5. 다차원 문서 포맷 확장 (MonoDoc & Raw Block) 

마크다운(Markdown)을 대체하고, 문서·데이터·스타일을 하나로 융합하는 다차원 확장 아키텍처입니다.

```mono
meta
	title	"MONO Architecture"

style
	body.p	size.14
	@alert	color.red

body
	p	"레벨 규칙에 따라 기저 스타일 적용"
	p@alert	"태그 규칙에 따라 red 오버레이 스타일 투영"

```

1. 
**내용과 스타일의 다차원 분리 (Multi-dimensional Projection)** 


* 
**일반 규칙 (기저 레이어):** 노드의 깊이(Depth) 자체가 곧 디자인 규칙이 됨 (예: Depth 2는 무조건 15px).


* 
**태그 규칙 (오버레이 Layer):** `@alert` 등의 속성 태그를 통해 독립된 스타일 차원(`style.@alert`)의 좌표와 본문 노드를 **양자 얽힘(Link)** 처리하여 합성 렌더링합니다.




2. 
**원시 데이터 블록화 (Raw Block Absorption)** 


* OpenCV `Mat` 구조의 헤더/버퍼 분리 철학을 계승하여 대용량 매트릭스·이미지·테이블의 '차원의 저주(표기법 폭발)'를 해결합니다.


* 트리는 오직 차원 스펙(`shape.2x2`)만 명시하고, 수만 개의 픽셀/데이터는 **단일 원시 바이트/문자열 블록**으로 흡수하여 초고속 오프셋 인덱싱을 구현합니다.
